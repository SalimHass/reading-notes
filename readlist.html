Big O: Analysis of Algorithm Efficiency

Overview

The big O in any algorithm is to describe worst case scenario of the performance , the performance related to Time and Space. There are 4 key areas to analyze:

1- Input size
2-Unit of measurment
3- Orders of growth
4-Best case, Worst case and Average case

Input size

Input Size refers to the size of the parameter values that are read by the algorithm.its not just the number of parameters , it takes in consideration the size of each parameter value.
We will use the (n) to refer to the input size.

Units of Measurement

We need to quantify our measurement of time , to do that we should take in mind three aspects of time:
1-The time in milliseconds from the start of a function execution until it ends.
2-The number of operations that are executed.
3-The number of “Basic Operations” that are executed.
In order to quantify memory space , we need to take in consideration four sources of memory usage during function time-run: 1-The amount of space needed to hold the code for the algorithm.
2-The amount of space needed to hold the input data.
3-The amount of space needed for the output data. 4-The amount of space needed to hold working space during the calculation.

Order of Growth

We describe the order of the efficiency by referring to the input size in and measure the time and space required to execute the code for n input .

Linked Lists

A linked list is a sequence of Nodes that are connected to each other . The defining feature of linked lists is that each Node references the next Node in the link.

Terminology:

1- linked list : A data structure that contains nodes that links/points to the next node in the list.
2- Singly: the number of references the node has, this means there is one reference. The reference refers to the next node.
3-Doubly : This means there are two references within the node. and it means that there is a reference to both the next and previous node.
4-Node : Nodes are the individual items/links that live in a linked list. Each node contains the data for each link.
5-Next : Each node contains a property called Next ,This property contains the reference to the next node.
6-Head : a reference of type Node to the first Node in a linked list.
7-Current : The Current is a reference of type Node to the node that is currently being looked at. When traversing, you create a new Current variable at the Head to guarantee you are starting from the beginning of the linked list.
